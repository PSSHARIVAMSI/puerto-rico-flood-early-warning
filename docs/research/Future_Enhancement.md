# Pull Request Description — Future Enhancements Backlog from PR #14

## Overview

This pull request extracts noncritical enhancement ideas and future recommendations from PR #14 and organizes them into a structured backlog document. The objective is to improve scope control by separating core capstone deliverables from larger future enhancements and experimental ideas.

## Purpose

PR #14 included several valuable discussions and enhancement proposals that extended beyond the immediate project scope. Keeping these ideas embedded within the implementation PR increased complexity and made prioritization difficult.

This PR introduces a dedicated future-work backlog document that:

- Preserves important future ideas
- Keeps the current capstone focused and manageable
- Improves project organization and maintainability
- Provides a clearer roadmap for future contributors

## Backlog Categories

### 1. Keep for Current Capstone

Features and improvements that align directly with current deliverables and can realistically be completed within the capstone timeline.

### 2. Future Capstone

Enhancements identified as high-value future directions requiring additional research, infrastructure, or data integration. Examples include:

- **Terrain layer analysis integration** for improved geospatial and flood-risk modeling
- **Earthquake and seismic activity data integration**
- **Expanded LLM integration** to support a more advanced AI-powered chatbot and contextual risk explanations
- **Additional hazard-layer enrichment** and predictive modeling improvements

### 3. Nice-to-Have Only

Optional features intended primarily for visualization, explainability, or user experience improvements, including:

- A secondary granularity layer showing individual sensor contributions to the final risk score
- Additional dashboard customization and interactive filtering
- Advanced visualization enhancements and exploratory analytics features

## Benefits

- ✓ Improves scope management and project focus
- ✓ Reduces implementation complexity in large PRs
- ✓ Preserves future development ideas in a structured format
- ✓ Provides clearer prioritization for contributors and reviewers
- ✓ Creates a roadmap for future iterations of the project

## Deliverable

Added a concise backlog/future-enhancements note documenting deferred ideas and categorizing them by implementation priority, scope relevance, and future research potential.
